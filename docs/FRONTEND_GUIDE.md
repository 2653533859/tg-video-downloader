## 🚀 使用流程

### 1. 启动应用

```bash
# 确保已登录 Telegram
python3 login.py

# 启动应用
python3 app.py

# 或使用 Docker
docker-compose up -d
```

### 2. 访问界面

打开浏览器访问：`http://localhost:5000`

### 3. 操作步骤

#### 第一步：检查连接状态
- 右上角显示 **"已连接"**（绿色）表示 Telegram 已连接
- 显示当前登录的用户信息

#### 第二步：选择频道
- 左侧栏显示所有频道/群组列表
- 点击任意频道查看视频
- 可使用搜索框过滤频道

#### 第三步：浏览视频
- 中间区域显示选中频道的视频列表
- 每个视频显示：
  - 缩略图（如果有）
  - 文件名
  - 文件大小
  - 视频时长
  - 下载按钮

#### 第四步：下载视频
- 点击视频卡片上的 **"下载"** 按钮
- 右侧下载面板会出现下载任务
- 实时显示下载进度

#### 第五步：管理下载
- 右侧面板显示所有下载任务
- 每个任务显示：
  - 文件名
  - 状态（排队中/下载中/已完成/失败）
  - 进度条
  - 下载速度
  - 已下载/总大小
- 可取消正在进行的下载
- 可重试失败的下载
- 可清除已完成的任务

## 🔍 功能详解

### 频道列表（左侧栏）

**功能**：
- 显示所有可访问的频道和群组
- 点击刷新按钮重新加载
- 搜索框可过滤频道名称
- 选中的频道高亮显示

**API**：`GET /api/dialogs`

### 视频列表（主区域）

**功能**：
- 显示当前频道的视频消息
- 支持分页浏览（上一页/下一页）
- 搜索功能（按文件名搜索）
- 缩略图预览
- 显示视频元信息

**API**：
- `GET /api/videos?dialog_id=<id>&page=<n>` - 分页加载
- `GET /api/search?dialog_id=<id>&query=<q>` - 搜索
- `GET /api/thumb/<msg_id>` - 缩略图

### 下载管理（右侧栏）

**功能**：
- 实时显示所有下载任务
- 自动轮询更新状态（每秒）
- 进度条动画
- 速度和大小显示
- 任务控制（取消/重试）
- 清除已完成任务

**API**：
- `POST /api/download` - 开始下载
- `POST /api/cancel` - 取消下载
- `POST /api/retry` - 重试下载
- `GET /api/download_status` - 获取状态

## 🎛️ 自定义配置

### 修改颜色主题

编辑 `static/css/style.css` 中的 CSS 变量：

```css
:root {
    --bg-warm-paper: #F6F1E8;    /* 背景色 */
    --accent-amber: #C8853F;     /* 强调色 */
    --text-ink: #1F2421;         /* 文本色 */
    /* ... 其他变量 */
}
```

### 修改轮询间隔

编辑 `static/js/app.js`：

```javascript
// 默认 1 秒
state.pollInterval = setInterval(updateDownloadStatus, 1000);

// 改为 2 秒
state.pollInterval = setInterval(updateDownloadStatus, 2000);
```

### 修改每页视频数量

后端修改 `app.py` 中的分页逻辑。

## 🐛 常见问题

### 1. 页面空白或样式错误
**原因**：静态文件路径错误  
**解决**：确保 `templates/` 和 `static/` 目录与 `app.py` 同级

### 2. 频道列表为空
**原因**：Telegram 未连接或未加入任何频道  
**解决**：
- 检查右上角连接状态
- 确保已运行 `login.py` 登录
- 在 Telegram 应用中加入一些频道

### 3. 视频无缩略图
**原因**：部分视频没有缩略图  
**解决**：正常现象，会显示占位符

### 4. 下载速度不显示
**原因**：刚开始下载时速度为 0  
**解决**：等待几秒后会显示实际速度

### 5. 下载任务不更新
**原因**：轮询被阻止或后端错误  
**解决**：
- 刷新页面
- 检查浏览器控制台错误
- 查看后端日志
